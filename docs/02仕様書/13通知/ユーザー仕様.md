# 通知 ユーザー仕様

最終更新: 2026-07-17
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 目的

Slack通知は、タスクのステータス変更とコメントのメンションを、関係者が画面外でも把握できるようにするために使います。通知先は設定したチャンネルであり、メンション対象者へのSlack DMではありません。

プロジェクト設定では、最初に次のどちらかを選びます。

- Slackアプリ方式：Incoming Webhookを1本使います。装飾と文面はこのアプリで管理でき、複数人のメンションも1件の投稿へまとめます。Slackアプリの作成・社内申請が可能な場合に向いています。
- Workflow Builder方式：Slack標準のWorkflowを使います。カスタムSlackアプリを申請しにくい場合に向いています。ステータス用とメンション用の2本を作り、装飾と本文はSlack側で設定します。

## ステータス変更通知

- 看板カードのドラッグ&ドロップでステータスを変更したとき。
- タスク詳細ドロワーでステータスを変更して保存したとき。
- リストビューのインライン操作でステータスを変更したとき。
- 進捗を100%に変更し、保存時に完了ステータスへ移行したとき。

同じステータス列内の並び替えや、子タスクの変更に伴う親タスクの自動ロールアップは通知しません。

## コメントメンション通知

- プロジェクト設定で「コメントのメンションを通知」がONの場合、コメントまたは返信に有効なメンバーのメンションが1件以上含まれると通知します。
- Slackアプリ方式では、複数人を1件の通知へまとめます。Workflow Builder方式では、SlackメンバーID登録済みの対象者ごとに1件ずつ通知します。
- メンション対象MemberIdはコメント保存時にサーバーで再検証します。存在しないメンバーだけが指定されている場合は通知しません。

## 通知内容と送信文面

- Slackアプリ方式には、ステータス変更用とコメントメンション用の個別のデフォルト文面があります。
- Slackアプリ方式ではプロジェクト設定の「Slack通知」で文面を編集し、「デフォルトに戻す」で初期文面へ戻せます。Workflow Builder方式ではSlackの「チャンネルにメッセージを送信」ステップで本文と装飾を編集します。
- ステータス変更文面では `{taskName}`, `{parentPath}`, `{beforeStatus}`, `{afterStatus}`, `{actorName}`, `{webAppUrl}` を使えます。
- メンション文面では `{taskName}`, `{parentPath}`, `{mentionedUsers}`, `{mentionedNames}`, `{actorName}`, `{commentText}`, `{webAppUrl}` を使えます。`{mentionedUsers}` はSlackメンバーID登録済みの対象者を本人メンションにします。
- `{commentText}` はSlack通知用に先頭1200文字まで差し込まれます。コメント本体の保存内容は省略されません。
- 文面は2000文字以内です。未対応のプレースホルダーを含む場合は保存できません。

## 通知されない操作

- タスク名、担当者、優先度、説明、日付だけの変更。
- カスケードリスケジュールによる日程変更。
- タスク追加・削除。
- メンションを含まないコメント・返信。
- 「コメントのメンションを通知」がOFFの場合のコメント・返信・メンション。

## 未設定時

- 選択中の方式で、ONにした通知に必要なWebhook URLが未設定の場合、その設定は保存できません。
- 未設定でもアプリの保存操作や画面表示は変わりません。

## 設定と確認

- プロジェクト設定は左側の縦タブから「Slack通知」を選び、右側で通知方式と必要なWebhook URLを設定します。
- ステータス変更通知とコメントメンション通知のON/OFF、方式・用途ごとの接続テスト、連携解除を同じ画面で操作できます。
- Webhook URL本体は保存後にブラウザへ返しません。ただし各URL欄には「設定済み／未設定」と伏せ字の保存状態が常に表示されます。入力欄が空でも設定済み表示があれば保存されています。空欄のまま再保存してもURLは消えません。
- 保存中と接続テスト中は、操作したボタンと近くの状態欄にスピナーが表示されます。
- Workflow Builder方式では、ステータス用とメンション用で画面に表示された変数をSlackへ登録します。`mentioned_user_id` だけは「ユーザーID」型、ほかは「テキスト」型です。単一の `{text}` 変数は使いません。
- 最終送信成功日時、最終失敗日時、HTTP応答コード、失敗理由を設定画面で確認できます。
- Slack送信が失敗してもタスク保存やコメント保存は取り消されません。
