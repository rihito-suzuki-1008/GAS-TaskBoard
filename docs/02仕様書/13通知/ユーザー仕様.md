# 通知 ユーザー仕様

最終更新: 2026-07-02
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 目的

Slack通知は、タスクのステータス変更を関係者が画面外でも把握できるようにするために使います。

## 通知される操作

- 看板カードのドラッグ&ドロップでステータスを変更したとき。
- タスク詳細ドロワーでステータスを変更して保存したとき。

## 通知内容

- タスク名。
- 親パス。
- 変更前のステータス列と変更後のステータス列。
- 実行者名。
- WebアプリURL。

## 通知されない操作

- タスク名、担当者、優先度、説明、日付だけの変更。
- カスケードリスケジュールによる日程変更。
- タスク追加・削除。
- コメント投稿・返信・メンション。

## 未設定時

- Slack Incoming Webhook URL が未設定の場合、通知は送信されません。
- 未設定でもアプリの保存操作や画面表示は変わりません。
