## 設計原則

- `WBS` は一方向の出力アーティファクトです。`HEADERS`, `ensureSchema_`, `loadAll` には含めません。
- `exportWbs()` は `withLock_` を使いません。書式出力中に他ユーザーの保存をブロックしないためです。
- 二重実行はクライアントのボタン状態と、Script Propertiesのトークン付き15分ガードで抑止します。ガードの取得・解放だけを短い `withLock_` 内で行い、長時間のシート書式出力中は保存ロックを保持しません。
- 出力の成功・失敗と経過時間をApps Script実行ログへ記録します。

## モデル構築

- `buildWbsModel_()` はGAS APIに触らない純粋関数です。
- `IncludeInWbs=false` のノードはサブツリーごと除外します。
- `DraftOwner` が設定された名称確定前ノードは、WBS番号・期間・進捗の計算前に除外します。
- WBS番号、最大階層、テンプレ列座標、日付範囲、値行列、背景色行列などをモデル内で確定します。
- 親ノードの進捗率とロールアップ期間は、非表示タスクを含むアプリ上の真の派生値を使います。
- 日付列は計画min/maxの前後7日を出力し、400日を超える場合は切り詰めます。

## テンプレートレイアウト

- 出力範囲は左側 `A:R` と日別ガント `S:` で構成します。
- `G1` 固定相当として、固定列は `A:F`、固定行はなしです。
- 行 `1:2` はプロジェクト名、開始日、終了日、更新日です。
- 行 `4:5` は2段ヘッダーです。`S4:` に日付値、`S5:` に曜日を出力します。
- 行 `6:10` はマイルストーンエリアです。見出し行は `B6`、日付列には名称と `▼` を上下に出します。隣接日付のマイルストーンは3レーンで縦にずらします。
- 行 `11:14` は会議体エリアです。見出し行は `B11`、本文は `C列` に `名称 + スケジュール` を出します。`ScheduleRuleJson` の毎日/毎週/隔週/月次ルール、または旧データのスケジュール/備考内の `YYYY-MM-DD` と `毎日` / `毎週◯曜日` を日付列の `▼` へ展開します。日付グリッド側に会議名は出しません。
- 行 `15` 以降がWBS本体です。深さ1は濃灰色のセクション行、深さ2以降は明細行です。
- タスク名は `B:E` の細い階層列へ深さ別に配置し、`F` は横溢れ表示用の空き列として使います。深さ5以降は `E列` に集約します。
- 管理列は固定座標です。`G`成果物、`H`備考、`I`会社名、`J`責任者、`K:L`計画日、`M`計画日数、`N:O`実績日、`P`実績日数、`Q`進捗率、`R`完了フラグ、`S:`ガントです。

## 実績日

- `deriveActuals_()` はActivityLogから実績日を導出します。
- `ChangedAt` はUTC ISOで保存しますが、WBS上の実績日・更新日は `Asia/Tokyo` 基準の日付へ変換します。
- 実績開始日は最初のステータス変更、または最初のprogress>0変更の早い日です。
- 実績終了日は現在の実効進捗が100%の場合のみ、`NewValueIsDone=true` の最後のログ日です。
- Nodesの `ActualStartDate` と `ActualEndDate` が有効な期間として両方設定されている場合は、その期間をActivityLog由来の日付より優先します。
- 手動実績期間は現在の進捗が100%未満でも保持・出力します。これはステータス履歴の補正値をユーザーが明示したものとして扱うためです。
- 日付範囲の算出には計画日だけでなく、最終的に採用した実績日も含めます。

## シート書き込み

- 一時ステージングシートへ全内容・書式を出力し、成功後に既存 `WBS` をバックアップ名へ変更してからステージングシートを `WBS` へ切り替えます。
- 切替完了後にバックアップを削除します。切替途中で失敗した場合はステージングを削除し、旧 `WBS` を元の名前へ戻します。
- この方式では出力途中の半端なWBSを利用者へ見せません。一方、出力成功ごとにWBSのsheetId（gid）は変わるため、外部からgid固定で参照しない運用とします。
- `setValues` と `setBackgrounds` は2D配列で一括実行します。
- `setValues` 前にWBS番号、タスク名、成果物、備考、会社名、責任者、マイルストーン/会議体のガント表示行へテキスト書式 `@` を設定します。
- 進捗列には、明細行のみ `0.0/0.15/0.3/0.45/0.6/0.75/0.9/1` のデータ検証を設定します。濃灰色のセクション行にはプルダウンを出しません。表示形式はテンプレートに合わせて `General` です。選択値ごとの見た目は、進捗列の条件付き書式で色分けします。
- 親タスクの派生進捗が8段階以外になる場合は、データ検証の選択肢のうち現在値以下で最大の値へ丸めて出力します。例: 96%相当は90%、100%ちょうどは100%です。
- `M`, `P`, `R`, `S:` の明細行には数式を出力します。
- 計画ガント帯、土日色、完了行の左側グレーアウトは条件付き書式で設定します。計画期間の緑は土日には適用せず、土日の赤/青が必ず見えるようにします。
- グリッド線は非表示、基本の列幅はテンプレートの見た目に合わせて固定px指定します。
- 会社名と責任者列は、出力文字数から算出した最小・最大幅付きの列幅を使います。明細セルだけ折り返しを有効にし、長い値は列幅の上限到達後に行高を広げて表示します。
