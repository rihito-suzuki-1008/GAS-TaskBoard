# WBS出力 ユーザー仕様

最終更新: 2026-07-16
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 概要

トップバーの「WBSに上書き」から、スプレッドシートの `WBS` シートを自動生成します。

## 実行条件

- 未同期の保存がある間は実行できません。
- 実行前に確認モーダルを表示します。
- 別の利用者が出力中の場合は二重実行せず、しばらく待って再試行するよう案内します。
- 完了すると「WBSシートを更新しました」と表示します。
- 日付範囲が400日を超える場合は、400日で切り詰めた警告を表示します。

## 出力内容

- シート上部にプロジェクト名、開始日、終了日、更新日を出します。
- テンプレート形式のメタ情報、マイルストーン、会議体、WBS表、日付グリッドを出力します。
- 成果物、備考、会社名、責任者、計画日、実績日、進捗率、完了フラグを出力します。
- 計画期間は淡緑のバー、実績期間は `★`、マイルストーンは名称と `▼`、会議体は左側の開催情報と日付列の `▼` で表示します。
- 近い日付のマイルストーンは縦位置をずらして表示します。
- 会議体の開催ルールが毎日/毎週/隔週/月次指定の場合、その開催日に `▼` を表示します。日付グリッド側には会議名を書きません。
- 深さ1のタスクはセクション行として濃灰色で表示します。

## WBS対象外

- タスク詳細の「WBSに反映」をOFFにしたタスクは、子タスクごと出力されません。
- 親がWBS対象外の場合、子がONでも出力されません。
- 名称入力が完了していない作成途中タスクは出力されません。
- アプリ内の看板・リスト・ガントでは、対象外タスクに「WBS外」チップを表示します。

## 注意

- アプリは `WBS` シートを読みません。
- `WBS` シートを直接手編集しても、次回出力で上書きされます。
- 出力は一時シートへ最後まで生成してから `WBS` と入れ替えます。生成途中で失敗した場合は、直前の `WBS` を残します。
