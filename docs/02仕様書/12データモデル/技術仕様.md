# データモデル 技術仕様

最終更新: 2026-07-21
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## シート一覧

- `Nodes`: ノード階層、ステータス、担当、計画日程、手動実績期間、説明、WBS項目、手動進捗、並び順、更新情報、削除情報。
- `Members`: メンバーID、名前、メール、色、会社名、SlackメンバーID。
- `StatusColumns`: 列ID、名称、並び順、完了列フラグ、色、親ロールアップ用の進行中列フラグ。
- `Dependencies`: 依存ID、先行ノードID、後続ノードID。
- `Comments`: コメントID、ノードID、投稿者、投稿時刻、本文、返信先、メンション宛先。
- `ActivityLog`: ステータス・進捗変更ログ。
- `Milestones`: WBS出力用マイルストーン。
- `Meetings`: WBS出力用会議体。
- `CalendarOverrides`: ガント表示用の案件共通カレンダー例外。
- `NotificationReads`: コメントメンション通知のメンバー別既読記録。

## ヘッダー定義

- ヘッダーは `src/00_Config.js` の `HEADERS` に定義します。
- 既存シートの列数が不足している場合は `ensureSheet_()` が不足列を追加します。
- Commentsの `ParentCommentId` / `Mentions` は末尾追加列です。既存行の空値は、親コメント・メンションなしとして扱います。
- Nodesの `Deliverable` / `Note` / `Progress` / `IncludeInWbs` / `DraftOwner` / `DraftExpiresAt` / `ActualStartDate` / `ActualEndDate` は末尾追加列です。Progress空欄は未完了0%、完了列100%の従来挙動へフォールバックします。
- `ActualStartDate` / `ActualEndDate` は手動実績期間です。両方空欄ならActivityLogから自動算出し、片方だけの値や不正な範囲は保存APIで拒否します。
- `DraftOwner` は名称未確定の作成途中ノードの所有MemberId、`DraftExpiresAt` は自動整理期限のISO日時です。通常ノードでは両方空欄です。
- Membersの `Company` / `SlackUserId` は末尾追加列です。SlackUserIdは本人へのSlackメンション通知に使う任意の固定IDで、API経由の重複登録を禁止します。
- StatusColumnsの `IsInProgressColumn` は末尾追加列です。既存行の空欄は従来名による互換判定を行い、列設定の保存時に明示値へ正規化します。
- Meetingsの `ScheduleRuleJson` / `StartDate` / `EndDate` は末尾追加列です。既存行の空値は、従来の `Schedule` 文字列による互換解釈へフォールバックします。
- CalendarOverridesの列は `Date` / `DayType` / `Name` です。「標準」は行として保存せず、既存の例外行を削除して自動判定へ戻します。
- NotificationReadsの列は `NotificationKey` / `RecipientMemberId` / `NotificationType` / `SourceId` / `ReadAt` です。未読イベントは複製保存せず、Comments.Mentionsと既読行の差分から動的に求めます。
- `WBS` は出力専用シートのため `HEADERS` に含めません。
- `Tags` は現在アプリでは読書きしませんが、既存列位置維持のため残します。
- ヘッダー自動移行は、既知の末尾列追加だけを許可します。並び替えや未知列は `SCHEMA_MISMATCH` として停止します。

## テキスト書式

- 日付・ID・日時列は `TEXT_COLUMNS` に定義します。
- `applyTextFormats_()` が対象列へ `@` 書式を設定します。
- アプリデータの日付はDate型ではなく `YYYY-MM-DD` 文字列です。
- Commentsの `Timestamp`, `ParentCommentId`, `Mentions` もテキスト書式で保持します。
- Meetingsの `ScheduleRuleJson`, `StartDate`, `EndDate` もテキスト書式で保持します。
- CalendarOverridesの全列もテキスト書式で保持します。
- NotificationReadsの全列もテキスト書式で保持します。
- Nodesの `DraftOwner`, `DraftExpiresAt`, `ActualStartDate`, `ActualEndDate` もテキスト書式で保持します。
- WBSシートだけは日付値と表示書式を使います。

## ソフトデリート

- Nodesは `DeletedAt` / `DeletedBy` を設定して削除扱いにします。
- 復元時は対象サブツリーの `DeletedAt` / `DeletedBy` を空に戻します。
- 読み込み系は `activeNodes_()` でDeletedAtが空のノードだけを対象にします。
- CommentsとDependenciesの関連行は物理削除しません。

## クライアント形式

- サーバー列名はPascalCase、クライアント形式はcamelCaseです。
- 変換は `clientNode_()`, `clientMember_()`, `clientStatusColumn_()`, `clientDependencies_()`, `clientMilestone_()`, `clientMeeting_()`, `clientComment_()` が担当します。
