# 看板ビュー ユーザー仕様

最終更新: 2026-07-24
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 目的

看板ビューは、ステータス列ごとにタスクカードを並べ、作業状態を直感的に把握するための画面です。

## レイアウト

- 担当者一覧は初期状態では畳まれ、フィルタ行の「担当者」から必要なときだけ表示できます。
- 上部の共通フィルタ行に検索、「自分のタスク」、適用中の条件、「＋フィルタ」があります。
- 「親も表示」、担当者一覧の表示切替、「列を追加」もフィルタ行の右端にあります。
- 中央にはステータス列が横並びで表示されます。
- 各列のヘッダーには列名、件数、完了列バッジ、列編集ボタン、タスク追加ボタンがあります。
- タスクが多い列は列の中を縦スクロールします。カード自体は縮まず、タスク名が常に読める高さを保ちます。
- 各ステータス列内のカードは、リストビューで階層を展開したときと同じ親子順で並びます。フィルタやステータスの違いで一部が表示されない場合も、残ったカード同士の相対順は維持されます。

## タスク操作

- カードをクリックすると右側の詳細ドロワーが開きます。
- カードを別ステータス列へドラッグ&ドロップすると、ステータスが変更されます。
- 列の空き領域にある追加ボタンから、その列のタスクを追加できます。
- 「親ノードも表示」をオンにすると、leafだけでなく親タスクもカードとして表示します。

## 列操作

- 列ヘッダーのドラッグハンドルをドラッグすると、列の左右順を変更できます。
- 列編集から名称、色、完了列設定を変更できます。
- 列追加から新しいステータス列を作成できます。

## 色の初期値

- 未着手: グレー。
- 進行中: 青。
- 完了: 緑。
- 色はプリセットから選べます。
- 自由色は詳細オプションにあります。
