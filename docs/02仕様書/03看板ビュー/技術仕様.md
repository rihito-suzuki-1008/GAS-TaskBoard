## 関連ファイルと関数

- `src/ClientRenderViews.html`: `renderKanban()`, `renderKanbanColumn()`, `renderCard()`, `renderMemberRail()`。
- `src/ClientBindings.html`: `bindKanban()`, `isBeforeColumn()`, `clearColumnDropIndicators()`。
- `src/ClientActions.html`: `saveNodePatch()`, `reorderStatusColumn()`。
- `src/03_StatusApi.js`: ステータス列保存と削除。

## カードD&D

- カードは `draggable="true"` と `data-card-id` を持ちます。
- dragstartで `state.draggedNodeId` と `text/plain` を設定します。
- 列bodyは `data-drop-column` を持ち、drop時に `saveNodePatch(nodeId, { statusColumnId })` を呼びます。
- optimisticApplyでローカルの `statusColumnId` を即時更新します。

## 列D&D

- 列ヘッダーは `data-column-drag` を持ちます。
- ドロップ位置の左右判定はポインタ位置が列幅の半分より左か右かで行います。
- 新しいSortOrderは前後列の中間値、端の場合は1000単位で採番します。
- 保存失敗時は旧SortOrderへ戻します。

## 表示対象

- `filteredNodes()` の結果からルートを除外します。
- 通常はleafのみ表示し、 `state.showGroupsInKanban` がtrueなら親ノードも表示します。
- 担当者フィルターは左レールのボタンで `state.filters.assigneeId` を更新します。
- `state.kanbanMemberRailCollapsed` の初期値はtrueです。担当者レールは共通フィルタ行の表示切替から開きます。
- `renderFilterViewActions()` が「親も表示」、担当者レール切替、「列を追加」を共通フィルタ行の右端へ描画し、看板内の独立ツールバーは置きません。

## ステータス色

- サーバーは `normalizeStatusColor_()` で色を正規化します。
- 未指定時は `defaultStatusColor_()` が名称と完了列から既定色を返します。
- クライアントはCSS変数 `--status-color` で列色を反映します。
