# AIエージェント連携 ユーザー仕様

最終更新: 2026-07-22

## 対象

ターミナルまたはModel Context Protocol（MCP）を利用できるAIエージェントから、本人のGoogle Workspaceアカウントとして案件のタスクを参照・追加・更新・コメント投稿できる。

## 基本動作

- AIエージェントは専用CLIまたはMCPサーバーを使用する。
- 初回利用時は専用のChromeプロファイルで既存のTaskBoard Webアプリを開き、利用者本人が通常どおりGoogleログインする。
- 標準利用ではGoogle Cloudプロジェクト、独自OAuthクライアント、Apps Script API実行デプロイを必要としない。
- タスク名などが複数候補に一致する場合、AIエージェントは勝手に対象を選ばずNodeIdの確認を求める。
- 競合が発生した場合、最新データを無断で上書きしない。
- スプレッドシートを直接編集せず、Webアプリと同じサーバーAPIを使用する。
- ローカルAIエージェントの操作中は、専用Chromeプロファイルのログインセッションを本人認証に使用する。

## 初期提供操作

- 案件・タスク・ステータス列・メンバー・依存関係の参照
- タスクの追加
- タスクの名称・ステータス・担当者・優先度・日程・進捗等の更新
- コメント投稿

削除・復元・依存関係の変更は、誤操作防止のため初期提供対象外とする。
