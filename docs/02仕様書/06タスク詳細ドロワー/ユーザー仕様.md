# タスク詳細ドロワー ユーザー仕様

最終更新: 2026-07-17
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 目的

タスク詳細ドロワーは、タスクの内容を編集し、コメントや依存関係も同じ場所で確認するための右側パネルです。デスクトップでは広い左側に名称・説明・コメント、右側にステータス等のプロパティを表示します。

## レイアウト

- 左側の主領域には、大きなタスク名、説明、コメントスレッド、コメント入力欄があります。
- 右側のプロパティ領域には、ステータス、担当者、優先度、進捗、予定、実績、WBS項目、関連タスク、依存関係があります。
- 左右は個別にスクロールできます。
- 画面幅が狭い場合は1列表示になり、主領域の下へプロパティが続きます。

## 開き方

- 看板カードをクリックします。
- リスト行のタスク名または編集ボタンをクリックします。
- ガントバーをクリックします。
- 子タスクshell作成後は自動で開きます。

## 編集項目

- 名称。
- ステータス。
- 優先度。
- 担当者。
- 開始日。
- 終了日。
- 実績着手日と実績終了日。
- 進捗率。末端タスクは0/15/30/45/60/75/90/100%から選べます。親タスクは子タスクから自動計算された値を表示します。
- 成果物。
- WBS備考。
- WBSに反映。
- 説明。

## 日付選択

- 開始日と終了日はアプリ内のカレンダーで選択します。
- 開始日を選ぶと、終了日が未設定または開始日より前の場合は同じ日に補正されます。
- 終了日は開始日より前の日を選べません。
- 「今日」で今日の日付を入れられます。
- 「クリア」で開始日と終了日をどちらも未設定に戻せます。
- 実績着手日と実績終了日は両方を設定します。片方を選ぶと、未設定のもう片方を同じ日で補います。
- 実績日の「クリア」は両方を「自動」へ戻します。自動時はステータス・進捗の変更履歴から算出します。
- 手動実績期間はWBSの実績日と `★` に反映されます。予定日やガントの日程、依存関係の自動調整には影響しません。

## 保存

- 「保存」ボタンで変更を保存します。
- 通常の保存は、保存ボタンを押した直後にドロワーを閉じ、書き込みはバックグラウンドで継続します。
- 名前が空の場合は保存できません。
- 保存中のタスクには小さな保存中表示が出ます。
- 競合やエラーはトーストで通知されます。
- 長いタスク名は名称欄とヘッダー内で折り返します。

## 補助機能

- 子タスクの期間に合わせるボタンで、親タスクの日程を子の最小開始日/最大終了日に合わせられます。
- タスク削除は確認ダイアログを経由します。
- 削除直後はトーストの「元に戻す」から復元できます。
- コメント投稿と依存関係追加/削除もドロワー内から行えます。

## WBS関連

- 「WBSに反映」をOFFにしたタスクは、子タスクごとWBS出力から除外されます。
- 親タスクがWBS対象外の場合、子タスク側のチェックは無効化されます。子タスク自身の設定値は保持されます。
- 成果物とWBS備考はWBSシート用の短い項目です。詳細な説明は従来どおり「説明」に記入します。
