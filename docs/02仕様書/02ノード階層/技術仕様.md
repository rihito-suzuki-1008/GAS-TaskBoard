# ノード階層 技術仕様

最終更新: 2026-07-04
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## データ構造

- NodesシートはType列を持ちません。
- 親子関係は `ParentId` で表現します。
- 子を持つかどうかは `childrenMap_()` と `computeDerived_()` で実行時に判定します。
- `clientNode_()` は `hasChildren`, `isLeaf`, `progress`, `displayStartDate`, `displayEndDate` をクライアントへ返します。

## 追加処理

- `addNode(payload)` は親ノードの存在をactive Nodesから検証します。
- `SortOrder` は同一親の兄弟末尾に `nextSortOrder_()` で採番します。
- shell作成時は `payload.isShell === true` の場合だけ空名を許可します。
- ステータス未指定時は先頭ステータス列を使います。

## 移動処理

- `moveNode(payload)` はルート移動、自己配下移動、存在しない親への移動を禁止します。
- 新しい親の兄弟末尾へ `SortOrder` を振り直します。
- 影響する祖先ノードもレスポンスに含め、進捗や表示期間を即時反映できるようにします。

## 削除と復元

- `deleteNode(payload)` は対象ノードと子孫の `DeletedAt` / `DeletedBy` を設定します。行は物理削除しません。
- 削除成功後、クライアントはUndo付きトーストを表示します。
- `restoreNode(payload)` は対象ノードと子孫の `DeletedAt` / `DeletedBy` を空に戻します。
- 親タスク自体が削除済みの子タスクだけを復元する操作は拒否します。
- 復元後は親ステータスロールアップを再実行し、対象サブツリーと祖先チェーンをレスポンスに含めます。

## 派生値

- `computeDerived_()` が親子関係をもとに進捗と表示期間を計算します。
- 完了列に属するleafは進捗100%、それ以外のleafは0%です。
- 親の進捗は子の平均です。
- 表示期間は自身の日程と子孫の日程を統合しますが、未設定ノードはロールアップ対象外です。
- `rollupParentStatuses_()` はノードの作成、保存、移動、削除後に直属の子ステータスを見て親ステータスを補正します。
- 直属の子がすべて完了列なら親も完了列へ、いずれかが「進行中」列なら親も進行中列へ寄せます。すべて完了ではなくても完了済みの子が1つでもあれば、親を進行中列へ寄せます。完了済みの親に未完了の子が加わった場合も、進行中列があれば親を進行中へ戻します。
