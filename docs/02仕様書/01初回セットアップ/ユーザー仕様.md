# 初回セットアップ ユーザー仕様

最終更新: 2026-07-16
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 起動条件

新品の案件スプレッドシートをWebアプリとして開き、Nodesにルートノードが存在しない場合、初回セットアップ画面が表示されます。途中の実行でMembersやStatusColumnsだけが作成済みでも、再試行できます。

途中データが検出された場合は、不足分から再開し、既存のメンバー・ステータス列を引き継ぐ案内を画面に表示します。

## 入力項目

- 案件名: ルートノード名として使われます。未入力の場合は「新規案件」です。
- 最初のメンバー名: 現在ログインしているユーザーの表示名として使われます。
- 表示色: メンバーのアバター色です。プリセットから選択できます。
- 詳細オプションでは自由なカラー指定もできます。

## 作成後にできること

- ルートノードが作成されます。
- 初期ステータス列として「未着手」「進行中」「完了」が作成されます。
- 自分自身が最初のメンバーとして登録されます。
- 以後は通常のタスク管理画面が表示されます。

## エラー時の見え方

- メールアドレスを取得できない場合はセットアップできません。
- 初期化が途中で失敗しても、再試行時は作成済みの自分のメンバーと初期ステータス列を再利用します。
- ルートノード作成済みの状態でMembersまたはStatusColumnsが欠けている場合は、不完全データとしてエラーになります。
- 再読み込みボタンから再試行できます。
