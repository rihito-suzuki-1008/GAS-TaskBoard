## スケール計算

- `ganttScale(rows)` は表示対象の `displayStartDate` と `displayEndDate`、マイルストーン日付、会議体に明示された日付、今日、作業日フィルタ範囲から全体範囲を決めます。
- `state.ganttZoom` に応じて固定余白を追加します。日表示は過去30日/未来120日、週表示は過去60日/未来240日、月表示は過去90日/未来365日です。
- `state.ganttZoom` に応じてbaseDayWidthを変えます。
- `state.ganttScaleFactor` を定義済みの段階値に丸めて乗算し、横の縮尺ボタンを反映します。
- ツールバーは全倍率ボタンを並べず、現在倍率と前後の倍率へ移動する「− / ＋」ステッパーだけを描画します。値は既存の `GANTT_SCALE_LEVELS` から選ぶため、スケール計算は変わりません。
- `ganttTreeWidth` / `ganttZoom` / `ganttScaleFactor` / `ganttTimelineCollapsed` はユーザー操作の確定時に表示設定へ保存し、次回起動時に検証して復元します。

## スクロール

- `state.ganttScrollLeft` / `state.ganttScrollTop` にガント右ペインのスクロール位置を保持します。
- `render()` 前に現在位置を取得し、再描画後に復元します。
- 初回ガント表示時は今日線が左から約1/4の位置に来るよう `ganttTodayScrollLeft()` で横位置を計算します。
- ツールバーの「今日へ」ボタンは同じ計算で右ペインを横スクロールします。
- 左側は `.gantt-left-body` だけを `translateY()` で同期し、ヘッダー `.gantt-axis-left` は `--color-surface-raised` の不透明背景とより高いstacking levelを持たせます。スクロールした行レイヤーがヘッダーの上へ描画されないよう、bodyとheaderを別のz-index層に固定します。
- 左側の予定レーンは `.gantt-left-timeline` として `.gantt-left-body` の外へ分離し、タスク行だけを `translateY()` します。右側の予定レーンは `.gantt-timeline-sticky` を `top: var(--gantt-axis-height)` で固定し、日付ヘッダー直下に残します。左右とも不透明背景とタスク行より高いstacking levelを持ち、スクロール中のタスクを下へ隠します。

## 作成・日程描画レーン

- タスク行の上に、`ganttTimelineRows(scale)` で作ったマイルストーン/会議体レーンを表示します。
- マイルストーンはラベル幅、会議体はコンパクトな `▼` 幅を見積もり、横方向に衝突する場合だけレーンを増やします。日/週/月や横縮尺により衝突判定は変わります。
- マーカー位置は日付列の左端ではなく、その日の中央です。
- `renderGanttTimelineMarker()` は `marker.tooltip` を専用ツールチップとして描画します。会議体のコンパクト表示でも、ホバー時に名称、日付、開催ルール、備考を確認できます。
- `state.ganttTimelineCollapsed` がtrueなら予定レーンは出力しません。
- 会議体は `ScheduleRuleJson` の構造化ルールを優先して、表示中の日付範囲へ展開します。毎日、毎週、隔週、毎月日付、毎月第n曜日/最終曜日を扱います。
- `ScheduleRuleJson` が空の旧データでは、名称/開催ルール/備考に含まれる `YYYY-MM-DD` と、開催ルールの `毎日` / `毎週◯曜日` をマーカー日付として扱います。
- 左ツリーにはリストビューと同じ「＋」を表示し、親ごとのゴースト行は表示しません。子shell作成は `createChildShell(parentId)` に統一します。
- 左ツリーのcollapse状態はリストと共通の `state.collapsed` を使い、localStorageの案件・ユーザー別表示設定へ保存します。
- 「＋」ホバー時はDOMだけの挿入ラインとチップを表示します。プレビューは保存キューやstateを変更しません。
- タイムライン背景は行の状態で動作を分けます。
  - 日付未設定のノード行は `data-gantt-schedule-node` を持ち、pointerupで `saveNodePatch(nodeId, { startDate, endDate })` を呼び、そのノード自身のバーを描きます。
  - 自身の日付が設定済みの子を持つノード行と空レーンは `data-gantt-create-parent` を持ち、pointerupで `createChildShell(parentId, { startDate, endDate })` を呼びます。
  - 日付設定済みの末端ノード行は背景ドラッグ対象にしません。
- 既存ノード行のクリック（ドラッグ距離が閾値未満）は何もしません。空レーンのみ、クリックでその日1日のルート直下shell作成を許可します。
- pointermove中は日境界にスナップしたプレビューと日付チップを表示します。
- Escキーでドラッグをキャンセルし、ドラッグ中は全体 `render()` を延期します。

## バー操作

- バーは `data-gantt-bar` を持ちます。
- 左ツリー行の `data-tree-node`、右行の `data-gantt-row-node`、バーの `data-gantt-bar` を同じnodeIdで照合し、pointerover/out時に `.gantt-linked-hover` を3要素へ同期します。
- バーの移動・左右ハンドル操作中は `state.ganttCreateDrag.type === 'bar'` を使ってpointeroutによる強調解除を抑止し、cleanup時に解除します。
- ハンドルは `data-handle="left"` / `data-handle="right"` で判定します。
- pointermove中はDOM styleだけを更新し、pointerupで `saveNodePatch()` を呼びます。
- 自身の日程がない親タスクは表示期間を持っていても、保存対象日程としては扱いません。

## 依存矢印

- 表示中の行にない依存先は、表示中の祖先にアンカーします。
- 折りたたみにより同じ表示ノード間へ集約される依存は1経路へまとめ、始点と終点が同じ表示ノードになる依存は描画しません。
- `ganttBarGeometry()` は日付幅だけでなくCSS上の最小幅24pxも含む実描画境界を返します。依存線の起点をバー内部へ置かないため、バー描画と経路計算で同じ境界を使います。
- `ganttDependencyRoutes()` は始点・終点間の全表示行を調べ、最も右へ伸びるバーより外側に縦レーンを確保します。行区間が重なる経路には別レーンを割り当てます。
- SVG pathは「先行バー右端→外側レーン→後続行の行間→後続バー左端」の直角折れ線です。合流・分岐ではバー端の接続位置を上下に分散します。
- 各経路は背景色の太いhaloと本線を重ねて描画し、線同士や日付グリッドとの交差を判別しやすくします。
