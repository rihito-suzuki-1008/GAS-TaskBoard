# 依存関係 ユーザー仕様

最終更新: 2026-07-02
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 目的

依存関係は、あるタスクが終わらないと別のタスクを開始できない、という前後関係を表します。

## 追加できる条件

- 依存元と依存先の両方が子を持たない末端タスクである必要があります。
- 両方に開始日と終了日が設定されている必要があります。
- 同じタスク同士の自己参照はできません。
- 同じ依存関係の重複追加はできません。
- 循環する関係は作れません。

## 操作場所

- タスク詳細ドロワーの依存関係パネルで追加・削除します。
- 先行タスクとして追加する操作と、後続タスクとして追加する操作があります。
- ガントビューでは依存関係が矢印で表示されます。

## 日程変更時の挙動

- 先行タスクの日程変更により後続タスクの開始が早すぎる場合、後続タスクは必要な日数だけ後ろへ移動します。
- もともと十分な余白がある場合は移動しません。
- 複数の先行タスクがある場合、最も遅い先行終了日を基準にします。
