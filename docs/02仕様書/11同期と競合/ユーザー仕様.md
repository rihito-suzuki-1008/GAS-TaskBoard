# 同期と競合 ユーザー仕様

最終更新: 2026-07-16
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 目的

このアプリはスプレッドシートを正本にしつつ、操作の体感速度を上げるためにブラウザにも直近データを保存します。

## ローカル保存

- 読み込み成功後、表示中データはブラウザのローカルストレージに保存されます。
- 次回起動時にスプレッドシートの読み込みが遅い場合でも、ローカルデータを先に表示できます。
- スプレッドシート同期に失敗しても、ローカル表示を継続する場合があります。
- キャッシュは案件とログインユーザーごとに分離され、6時間で期限切れになります。
- 設定画面から、このブラウザに保存されたキャッシュを削除できます。

## 自動同期

- タブが再び表示されたとき、ウィンドウへ戻ったときにスプレッドシート側の変更を確認します。
- 保存中の変更がある場合、自動同期は保留されます。
- 強制同期ボタンで手動同期できます。

## 保存中の挙動

- タスク追加、編集、削除、列変更、メンバー変更、コメント投稿、依存関係変更は、操作した瞬間に画面へ反映されます。
- スプレッドシートへの保存はバックグラウンドで行われます。
- 保存が終わる前でも、画面上では次の操作を続けられます。
- 同期表示が「スプレッドシートへ保存待ち」の間は、スプレッドシート確定待ちの操作があります。
- 「スプレッドシートへ保存待ち」の間にタブを閉じようとすると、保存未確定の操作があるためブラウザが確認します。
- 同一タスクへの連続保存は順番に処理されます。
- 保存応答の適用後に、まだ確定していない新しい操作を重ね直すため、古い確定値で新しい操作を上書きしません。
- 保存に失敗した場合はローカル表示を可能な範囲で戻し、エラーを通知します。
- 作成途中の名称未入力タスクは自分だけに表示され、再読み込み後も復元されます。24時間を過ぎた作成途中タスクは自動的にソフトデリートされます。

## 競合

- 他のユーザーが先に同じタスクを更新していた場合、競合として警告されます。
- 競合時は最新データが画面へ取り込まれます。
- 必要であれば内容を確認して再度編集してください。

## 2026-07-02の方針変更

- 以前は一部の操作だけが即時反映でした。
- 現在は、スプレッドシート通信待ちによるもっさり感を避けるため、保存系操作全体をローカルファーストへ寄せています。
- ただし、正本は引き続きスプレッドシートです。保存応答、強制同期、自動同期により正本へ収束します。
