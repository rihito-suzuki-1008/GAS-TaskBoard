# コメント ユーザー仕様

最終更新: 2026-07-17
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 目的

コメントは、タスクごとの連絡、判断履歴、補足情報を残すために使います。

## 表示

- タスク詳細ドロワー内にコメント欄があります。
- コメントには投稿者アバター、投稿者名、投稿日時、本文が表示されます。
- 親コメントの下に返信がインデント表示されます。返信への返信も同じ親コメントの返信として表示され、2階層目は作りません。
- コメント件数はタスクデータと一緒に保持され、表示に使われます。
- 自分宛のメンションを含むコメントは淡い背景で強調されます。

## 投稿

- コメント欄に本文を入力して投稿します。
- 空欄のコメントは投稿されません。
- 投稿後は一覧に即時反映されます。
- 各コメントの返信ボタンから返信モードに切り替えられます。キャンセルで通常投稿に戻ります。
- `@` を入力するとメンバー候補が表示され、選択すると本文に `@氏名` が挿入されます。
- プロジェクト設定でコメントメンション通知がONの場合、有効なメンションを含む投稿・返信は設定済みSlack Webhookのチャンネルへ通知されます。対象者へのSlack DMではありません。

## Markdown表示

- 太字、斜体、打ち消し、インラインコード、コードブロック、箇条書き、番号リスト、http/httpsリンク、単一改行に対応します。
- HTML、画像、見出し、テーブル、引用は表示変換しません。
- `javascript:` など http/https 以外のリンクはリンク化されません。

## 仕様上の制限

- 現時点でコメント編集はありません。
- 現時点でコメント削除はありません。
- 投稿者名は投稿時点のメンバー名で保存されます。後でメンバー名を変えても既存コメントの投稿者名は変わりません。
