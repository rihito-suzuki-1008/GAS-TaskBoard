最終更新: 2026-07-02
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 構成

- サーバーサイドはGoogle Apps Script V8です。
- フロントエンドはHTML Service上のVanilla JavaScriptです。
- データストアは同一スプレッドシート内の `Nodes` / `Members` / `StatusColumns` / `Dependencies` / `Comments` シートです。
- Webアプリは `doGet()` で `src/Index.html` を配信します。

## 分割ファイル

- `src/Code.gs`: Webエントリ、初期ロード、初回セットアップ、include、ping。
- `src/00_Config.js`: シート名、ヘッダー、テキスト書式、優先度定義。
- `src/01_NodeApi.js`: ノードCRUD、移動、日程フィット、ノードパッチ。
- `src/02_CollaborationApi.js`: コメント、メンバーCRUD。
- `src/03_StatusApi.js`: ステータス列CRUD。
- `src/04_DependencyApi.js`: 依存関係CRUD、DAG検証、連鎖リスケジュール。
- `src/05_Payloads.js`: レスポンス構築、クライアント用シリアライズ、派生値計算。
- `src/06_Sheets.js`: スキーマ保証、シート読み書き、テキスト書式。
- `src/07_Validation.js`: ロック、本人識別、入力正規化、ドメイン検証。
- `src/08_Utils.js`: 階層、日付、配列、文字列、IDなどの汎用関数。
- `src/Index.html`: HTMLシェルとテンプレートinclude。
- `src/Styles.html`: CSS。
- `src/Client*.html`: クライアントJSを役割別に分割。

## 絶対ルール

- 進捗率とガント拡張表示期間は保存しません。読み込み・保存のたびに `computeDerived_()` で計算します。
- 保存系APIは `LockService` で直列化します。
- ノード保存は `UpdatedAt` による楽観的競合検知を行います。
- Members、StatusColumns、Dependenciesはロックによる直列化のみで、UpdatedAt照合は行いません。
- 削除はソフトデリートです。
- 完了列は常に1件だけです。
- 日付は `YYYY-MM-DD` 文字列として扱います。
