# 全体像 技術仕様

最終更新: 2026-07-22
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 構成

- サーバーサイドはGoogle Apps Script V8です。
- フロントエンドはHTML Service上のVanilla JavaScriptです。
- データストアは同一スプレッドシート内の業務データシートです。自分宛メンションの既読状態は `NotificationReads` に保存します。
- Webアプリは `doGet()` で `src/Index.html` を配信します。

## 分割ファイル

- `src/Code.js`: Webエントリ、初期ロード、初回セットアップ、include、ping。
- `src/00_Config.js`: シート名、ヘッダー、テキスト書式、優先度定義。
- `src/01_NodeApi.js`: ノードCRUD、移動、日程フィット、ノードパッチ。
- `src/02_CollaborationApi.js`: コメント、メンバーCRUD。
- `src/03_StatusApi.js`: ステータス列CRUD。
- `src/04_DependencyApi.js`: 依存関係CRUD、DAG検証、連鎖リスケジュール。
- `src/05_Payloads.js`: レスポンス構築、クライアント用シリアライズ、派生値計算。
- `src/06_Sheets.js`: スキーマ保証、シート読み書き、テキスト書式。
- `src/07_Validation.js`: ロック、本人識別、入力正規化、ドメイン検証。
- `src/08_Utils.js`: 階層、日付、配列、文字列、IDなどの汎用関数。
- `src/09_Notifications.js`: Slack通知の設定、組み立て、送信。
- `src/12_NotificationApi.js`: アプリ内通知の一覧、対象コメント取得、既読更新。
- `src/Index.html`: HTMLシェルとテンプレートinclude。
- `src/Styles.html`: CSS。
- `src/Client*.html`: クライアントJSを役割別に分割。

## 共通フィルタ

- クライアントの `state.filters` は `query` / `parentId` / `statusIds` / `assigneeId` / `priority` / `wbsInclusion` / `dateStart` / `dateEnd` を持ちます。
- `renderFilters()` は検索、「自分のタスク」、適用中のフィルタチップ、「＋フィルタ」、ビュー固有操作を1つのflex行へ描画します。
- `renderFilterControl()` は既存の `renderDropdown()` / `renderDatePicker()` をピル型トリガーとして再利用します。条件値と絞り込み判定は変更しません。
- 「＋フィルタ」で未適用の条件を選ぶと、一時トリガーとピッカーを表示します。値を選んだ後は適用中チップへ切り替え、外側クリックで値未選択の一時トリガーを閉じます。
- 作業日フィルタは `dateStart` と `dateEnd` を必ずペアで設定します。
- WBS反映フィルタの `wbsInclusion` は未指定時が空文字、対象が `included`、対象外が `excluded` です。判定にはノード自身の `includeInWbs` だけでなく、祖先の対象外設定も含む `isNodeWbsExcluded()` を使用します。
- `matchesFilters(node)` はノード自身の `startDate` / `endDate` と作業日範囲が重なるかで判定します。日付未設定ノードは作業日フィルタには一致しません。
- リスト/ガントのツリー表示では、フィルタ一致ノードの祖先も表示対象に含めます。

## 絶対ルール

- 進捗率とガント拡張表示期間は保存しません。読み込み・保存のたびに `computeDerived_()` で計算します。
- 保存系APIは `LockService` で直列化します。
- ノード保存は `UpdatedAt` による楽観的競合検知を行います。
- Members、StatusColumns、Dependencies、NotificationReadsはロックによる直列化のみで、UpdatedAt照合は行いません。
- 削除はソフトデリートです。
- 完了列は常に1件だけです。
- 日付は `YYYY-MM-DD` 文字列として扱います。

## パフォーマンス方針

- 2026-07-02以降、UI操作はローカルファーストの楽観的更新を基本にします。
- ブラウザは操作直後に `state` とローカルキャッシュを更新し、スプレッドシート保存はバックグラウンドで行います。
- 選択中ビューとガント表示設定は業務データキャッシュから分離したユーザー別表示設定へ保存します。
- 保存応答を受けたらサーバー計算済みの派生値・カスケード結果・最新一覧でローカル状態を収束させます。
- 保存失敗時は該当操作をロールバックし、同期状態をエラーにしてトースト通知します。
- スプレッドシートは正本のままです。ローカル表示は体感速度のための先行表示であり、整合性判定はサーバー側で維持します。
