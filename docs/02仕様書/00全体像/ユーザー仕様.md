# 全体像 ユーザー仕様

最終更新: 2026-07-21
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 目的

このアプリは、1つのGoogleスプレッドシートを1案件のデータストアとして使う社内向けタスク管理アプリです。ユーザーはスプレッドシートを直接編集せず、Webアプリ画面から看板、リスト、ガントを切り替えてタスクを管理します。

## 基本画面

- 上部には案件名、ビュー切替、同期ピル、通知ベル、メンバー管理、設定、WBS出力が表示されます。同期ピルをクリックすると最新の状態を確認できます。
- 通知ベルは自分宛のコメントメンションを開き、未読があると数字バッジを表示します。通知から該当タスクとコメントへ移動できます。
- ビューは「看板」「ガント」「リスト」の3種類です。
- フィルタ行には検索、「自分のタスク」、「＋フィルタ」が表示されます。「＋フィルタ」から階層、ステータス、担当者、優先度、期間を追加できます。
- 適用中の条件はチップで表示されます。チップをクリックすると条件を編集でき、「×」で解除できます。同じ条件をドロップダウンとチップへ二重表示しません。
- 検索は名称、説明、成果物、WBS備考、ステータス名、担当者名を対象にします。
- 作業日は単体日または日付範囲で指定でき、その期間と重なるタスクだけが表示されます。
- タスクの追加や編集は、右側から開くドロワー、または専用モーダルで行います。
- 保存状態は画面上部の同期表示で確認できます。

## 共通操作

- タスク名、担当者、ステータス、優先度、期間、説明はタスク詳細ドロワーで編集します。
- 日付選択はアプリ内のカレンダーで行います。ブラウザ標準のカレンダーは使いません。
- Escキーで開いているドロップダウン、モーダル、ドロワーを閉じられます。
- モーダルまたはドロワーを開いている間、Tabキーのフォーカスはその画面内を循環します。
- 未保存のshellタスクを空のまま閉じると、自動的に削除されます。
- 保存失敗や競合が起きた場合はトースト通知で理由が表示されます。

## 表示ルール

- 親タスクの進捗率は子タスクから自動計算されます。
- 親タスクのガント表示期間は子孫の期間から自動計算されます。
- 削除済みタスクは通常画面に表示されません。
- 初期ステータスは左から「未着手」「進行中」「完了」です。
- デフォルト色は、未着手がグレー、進行中が青、完了が緑です。

## 注意事項

- ブラウザには直近データがローカル保存されますが、正本はスプレッドシートです。
- 複数人が同時に編集している場合、古い画面から保存しようとすると競合として扱われることがあります。
- 依存関係があるタスクの日程を変更すると、後続タスクが自動的に繰り下がる場合があります。
