# リストビュー ユーザー仕様

最終更新: 2026-07-22
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 目的

リストビューは、階層構造、担当者、ステータス、優先度、期間、進捗を表形式で確認する画面です。

## 表示項目

- 名称。
- ステータス（看板列と同じ色のチップ。クリックすると切り替えられます）。
- 担当者。
- 優先度。
- 期間。
- 進捗率。末端タスクはクリックすると色付き候補から編集できます。親タスクは読み取り専用です。
- 完了列の末端タスクは100%固定で、進捗選択は無効です。差し戻す場合は先にステータスを完了列以外へ変更します。
- 子タスク追加ボタン。

## 直接編集

- 名称、ステータス、担当者、優先度、期間、進捗率は表の中で直接編集できます。
- ステータス、担当者、優先度、期間、進捗率は通常時はチップや日付表示として見せ、クリック後だけ選択UIを開きます。
- 選択UIは最下行や画面端から開いても見切れない位置へ自動調整されます。高さが足りない場合は選択UIの中だけをスクロールできます。
- 名称はフォーカスアウトまたはEnterで保存されます。
- 進捗率の色はWBS出力の進捗色と揃えます。

## 階層操作

- 子を持つ行は開閉できます。
- 開閉状態はガントビューと共通です。同じブラウザでは案件・ログインユーザーごとに保存され、次回起動時にも復元されます。
- 行左のハンドルをドラッグして、別の行の前/後/内側へドロップできます。
- 表示順はD&Dで変更します。名称・ステータス・優先度・進捗によるソートボタンは表示しません。
- 親タスクを移動すると子タスクも一緒に移動します。
- 子タスクを親より上の兄弟位置へ置いたり、別の親タスクの内側へ入れたりできます。
- 子タスク追加ボタンは行ホバー時に右端へ表示されます。
- 親ごとの「タスクを追加」行は常設しません。通常は行ホバーの追加ボタンから子タスクを作成します。
- 一覧末尾に、ルート直下へ追加するための「タスクを追加」行を表示します。フィルター中も消えません。
- フィルター結果が0件の場合は「タスクを追加」と「条件を解除」を併記します。追加したタスクが現在条件に一致しない場合は、保存後に一覧から隠れることがあります。

## 子タスク作成体験

- 子タスク追加ボタンにホバーすると、実際に挿入される位置へ半透明のプレビュー行が表示されます。
- 追加ボタンを押すと、行の直下にshellタスクがアニメーションで挿入されます。
- 右ドロワーが開き、名称欄が選択済みになります。
- 名前を入力して保存すれば通常タスクになります。
- 何も入力せず閉じたshellは自動削除されます。
