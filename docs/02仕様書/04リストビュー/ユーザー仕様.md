# リストビュー ユーザー仕様

最終更新: 2026-07-04
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 目的

リストビューは、階層構造、担当者、ステータス、優先度、期間、進捗を表形式で確認する画面です。

## 表示項目

- 名称。
- ステータス（看板列と同じ色のドロップダウン）。
- 担当者。
- 優先度。
- 期間。
- 進捗率。末端タスクは色付きドロップダウンで編集できます。親タスクは読み取り専用です。
- 子タスク追加ボタン。

## 並び替え

- ツールバーの「名称」「ステータス」「優先度」「進捗」で表示順を切り替えられます。
- 同じボタンを再度押すと昇順/降順が反転します。
- 階層の親子関係は維持されます。

## 直接編集

- 名称、ステータス、担当者、優先度、期間、進捗率は表の中で直接編集できます。
- 名称はフォーカスアウトまたはEnterで保存されます。
- 進捗率の色はWBS出力の進捗色と揃えます。

## 階層操作

- 子を持つ行は開閉できます。
- 行左のハンドルをドラッグして、別の行の前/後/内側へドロップできます。
- 親タスクを移動すると子タスクも一緒に移動します。
- 子タスクを親より上の兄弟位置へ置いたり、別の親タスクの内側へ入れたりできます。
- 子タスク追加ボタンは行ホバー時に右端へ表示されます。
- 親ごとの「タスクを追加」行は常設しません。通常は行ホバーの追加ボタンから子タスクを作成します。
- 一覧末尾にだけ、ルート直下へ追加するための「タスクを追加」行を表示します。

## 子タスク作成体験

- 子タスク追加ボタンにホバーすると、実際に挿入される位置へ半透明のプレビュー行が表示されます。
- 追加ボタンを押すと、行の直下にshellタスクがアニメーションで挿入されます。
- 右ドロワーが開き、名称欄が選択済みになります。
- 名前を入力して保存すれば通常タスクになります。
- 何も入力せず閉じたshellは自動削除されます。
