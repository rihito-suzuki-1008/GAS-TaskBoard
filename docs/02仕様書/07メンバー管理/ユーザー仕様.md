# メンバー管理 ユーザー仕様

最終更新: 2026-07-02
参照元: `docs/01要件定義・設計/要件定義書.md` / `docs/01要件定義・設計/基本設計書.md` / `DESIGN.md` / `AGENTS.md`

## 目的

メンバー管理では、タスク担当者として使う人の名前、メールアドレス、会社名、表示色を管理します。

## 開き方

- 上部の「メンバー」ボタンを押します。
- メンバー管理モーダルが開きます。
- 自分のメールアドレスが未登録の場合は、初回表示時に参加モーダルが開きます。「あとで」を選んだ場合も、未登録バナーから参加モーダルを再表示できます。

## 追加・編集

- 氏名、メール、会社名、色を入力します。
- 色はプリセットから選べます。
- 詳細オプションで自由色も選べます。
- 既存メンバーの編集ボタンを押すと入力欄へ内容が読み込まれます。

## セルフ登録

- 参加モーダルでは、ログイン中のメールアドレスが固定表示されます。メールアドレスは変更できません。
- 氏名と表示色を入力して「参加する」を押すと、自分自身をメンバーとして登録します。
- 「あとで」を押すと閲覧専用のまま使えます。保存操作、コメント投稿、担当者変更は利用できません。
- 同じメールアドレスで複数タブから参加しても、メンバーは重複作成されません。

## 削除

- メンバー行の削除ボタンから削除確認に進みます。
- 自分自身を削除する場合は追加の確認チェックがあります。
- 最後の1人は削除できません。
- 削除されたメンバーは担当中タスクから外れます。

## 表示への影響

- 担当者はアバターとして表示されます。
- 看板左の担当者一覧にも反映されます。
- タスク詳細の担当者選択肢にも反映されます。
- WBS出力では、担当者に設定された会社名が会社名列へ出力されます。
