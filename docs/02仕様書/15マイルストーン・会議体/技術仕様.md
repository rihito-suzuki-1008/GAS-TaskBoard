## データ

- `Milestones`: MilestoneId, Name, Date, Note, SortOrder。
- `Meetings`: MeetingId, Name, Schedule, Note, SortOrder, ScheduleRuleJson, StartDate, EndDate。
- どちらも `HEADERS` に含め、通常の `ensureSchema_()` 対象です。
- WBSシートとは異なり、アプリの正本データシートです。
- `Schedule` は人間向け表示ラベルです。自動展開は `ScheduleRuleJson` と `StartDate` / `EndDate` を正とします。
- `ScheduleRuleJson` は `type`, `interval`, `weekday`, `dayOfMonth`, `nth` などを持つJSON文字列です。
- `type` は `daily`, `weekly`, `monthlyDate`, `monthlyNth` を扱います。隔週は `weekly` + `interval: 2` です。
- 旧データ互換として、`ScheduleRuleJson` が空の場合は `Schedule` 内の `毎日` / `毎週◯曜日` と `YYYY-MM-DD` を引き続き解釈します。

## 保存

- APIは `withLock_()` と `requireCurrentMember_()` を使います。
- UpdatedAt照合は行いません。
- クライアント採番IDを受け取り、重複IDは拒否します。
- クライアントは `queueBackgroundMutation('milestones'...)` / `queueBackgroundMutation('meetings'...)` で楽観的更新します。
