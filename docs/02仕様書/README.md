## 更新ルール

- UIや操作手順を変えたら、該当機能のユーザー仕様を更新します。
- API、シート列、検証、同期、競合、派生値計算を変えたら、該当機能の技術仕様を更新します。
- 派生値を保存しない、カスケード前に最新シートを読み直す、完了列を1件に保つなどのアーキテクチャ原則は、仕様書より上位の制約として扱います。
