# リリース・運用チェックリスト

このアプリは社内向けの案件別タスク管理アプリです。1つのスプレッドシートを1案件の正本データストアとして扱い、`WBS` シートは出力専用の成果物として扱います。

## リリース前チェック

### 自動チェック

リリース前に以下を実行します。

```sh
node --check src/01_NodeApi.js
node --check src/04_DependencyApi.js
node --check src/05_Payloads.js
node --check src/10_WbsExport.js
for f in src/Client*.html; do node --check --input-type=commonjs < "$f" || exit 1; done
node --test tests/*.test.mjs
git diff --check
```

### 手動スモーク

- 初回セットアップ画面から案件を作れること。
- メンバー追加、会社名、色変更が保存されること。
- 看板でカードを別ステータスへドラッグできること。
- リストで名称、ステータス、担当、優先度、期間、末端進捗を直接編集できること。
- リスト/ガント/看板でタスク順と親子関係をD&Dで変更できること。
- 子タスクの一部が完了した親が進行中になり、子がすべて完了した親が完了になること。
- 依存関係追加時と先行タスクの日程変更時に、後続タスクが必要分だけ後ろへずれること。
- 詳細ドロワーの保存ボタン押下直後にドロワーが閉じ、保存失敗時はエラーが出ること。
- タスク削除後、トーストの「元に戻す」で削除したタスクと子タスクを復元できること。
- マイルストーン/会議体がガント上に表示され、会議体の `▼` ホバーで内容を確認できること。
- 100件前後のタスクで、保存、D&D、ビュー切り替え、WBS出力の体感が許容範囲であること。

## WBS出力スモーク

WBS出力後、実シートで以下を確認します。

- `WBS` シートが削除再作成ではなく既存gid維持で上書きされていること。
- A列のWBS番号が `1-1` のような値でも日付化していないこと。
- 進捗率プルダウンが明細行だけに出て、セクション行やヘッダー行には出ていないこと。
- 進捗率の色が 0/15/30/45/60/75/90/100 の選択肢に対応していること。
- 計画期間の緑より、土日の赤/青が見えること。
- マイルストーンが近接日でも縦にずれて表示されること。
- 会議体は日付グリッドに会議名を書かず、開催日の `▼` だけが出ること。
- 実績開始日/実績終了日がActivityLogから想定通りに出ること。
- 400日を超える範囲では警告が表示され、日付列が切り詰められること。

## 権限と配置

- Webアプリは「アクセスしたユーザーとして実行」で運用します。
- 同一Google Workspaceドメイン内のユーザー利用を前提にします。
- 利用者にはコンテナスプレッドシートへの編集権限が必要です。
- 直接シート編集は技術的には可能ですが、アプリ側のバリデーションを迂回します。通常利用者にはWebアプリから操作するよう案内します。
- 案件を増やす場合は、テンプレートのスプレッドシートをコピーし、案件ごとにGASプロジェクト/デプロイを運用します。

## デプロイ手順

```sh
clasp push -f
clasp deploy --deploymentId <production-deployment-id> -d "<release note>"
git status --short
git push origin main
```

デプロイ後はWebアプリをリロードし、トップバー表示、ビュー切り替え、保存1件、WBS出力1回を確認します。

## 復旧

- 誤削除はソフトデリートなので、通常画面からは消えますがNodesシート上には残ります。
- 削除直後はトーストの「元に戻す」で復元できます。
- 削除直後以外の復旧が必要な場合は、スプレッドシートの変更履歴、または管理者によるシートメンテナンスで対応します。
- `WBS` シートは出力専用です。シート上の手編集は次回出力で消えます。
- デプロイ後に重大な問題が出た場合は、直前の安定コミットへ戻して `clasp push -f` と本番deploymentIdへの再デプロイを行います。

## 既知の運用限界

- リアルタイム同期ではありません。保存タイミングで競合検知し、定期同期は未保存操作がない時だけ適用します。
- Nodes以外のシートはUpdatedAt照合を行わず、LockServiceによる直列化のみです。
- ActivityLogは追記専用です。WBS出力時だけ読み込みますが、長期運用で肥大化した場合は管理者が古い案件のログ退避を検討します。
- 主な想定規模は100タスク前後です。大幅に超える場合は、描画、WBS書式出力、google.script.runの体感遅延を再評価します。
