# AIレビュー依頼プロンプト

以下のリポジトリを、社内向けGoogle Apps Script製タスク管理アプリとしてレビューしてください。

## 前提

- このアプリは、1つのGoogleスプレッドシートを1案件のデータストアとして使うWebアプリです。
- フロントエンドはGAS HTML Service、バックエンドはApps Script、データストアは同一スプレッドシート内の各シートです。
- 主な機能は、看板・リスト・ガント、タスク階層、担当者、ステータス列管理、依存関係、コメント、進捗率、マイルストーン/会議体管理、WBS出力です。
- `WBS` シートは一方向の出力アーティファクトです。アプリは読み込みません。
- WBS出力の見た目・列幅・行高・色・罫線・条件付き書式・数式・プルダウンは、会社指定テンプレートの再現が目的です。テンプレートとしての体裁そのものへの大きな変更提案は控えてください。
- ただし、WBS出力のバグ、データ欠落、性能劣化、Apps Script上の失敗リスク、メンテナンス性の問題は遠慮なく指摘してください。
- アプリ本体は独自実装です。GAS/HTML Service/SpreadsheetAppで現実的に実現できる範囲なら、UX・設計・性能・安全性・運用性の改善提案を歓迎します。

## レビューで見てほしい観点

1. プロダクトとして社内利用に出せる完成度か
2. 実運用で壊れそうな不具合、データ不整合、競合、ロストアップデート
3. GAS/SpreadsheetApp特有の制限、実行時間、ロック、API呼び出し回数、クォータ面のリスク
4. UI/UXの詰めどころ。特に看板・リスト・ガント・ドロワー・プロジェクト設定
5. タスク階層、親ステータスロールアップ、進捗率、依存関係リスケジュールの仕様矛盾
6. WBS出力のテンプレート再現に関する実害のあるズレ
7. セキュリティ、権限、社内運用、監査ログ、復旧性
8. コード構成、責務分離、テスト不足、ドキュメント不足
9. 今すぐ直すべきもの、後回しでよいもの、やらなくてよいものの切り分け

## 出力形式

以下の順で、日本語で回答してください。

1. 総評: 社内プロダクトとして出してよいかを率直に判断
2. P0/P1/P2/P3の改善リスト
   - P0: リリース前に必ず直すべき致命的問題
   - P1: 早期に直すべき実運用リスク
   - P2: 使い勝手・保守性・性能の改善
   - P3: 将来検討でよい改善
3. 各指摘について、該当ファイル・関数・仕様書が分かる場合は具体的に示す
4. WBSテンプレートへの「好みのデザイン変更」提案は避け、実害のある問題に絞る
5. 追加テスト案
6. 最後に、次の1週間でやるべき現実的な改善計画

## 注意

- 既存の設計思想を尊重してください。特に `AGENTS.md` と `docs/02仕様書/README.md` の原則を先に読んでください。
- 単に「Reactへ移行」「DB化」「フレームワーク導入」のような大改修を主提案にしないでください。GAS上での改善を優先してください。
- 抽象論ではなく、このリポジトリの実装に即した指摘にしてください。
